# Rich Draft Editor — Design Spec

**Date:** 2026-05-07
**Status:** Approved

## Overview

Replace the plain `<s-text-area>` in the `DraftBlock` component with a rich text editor (TipTap) that supports the formatting operations useful for customer support emails: bold, italic, underline, lists, blockquote, hyperlinks, and clear formatting. The LLM continues generating Markdown, which is converted to HTML server-side before being stored and displayed.

## Scope

- **In scope:** `DraftBlock` component inside the inbox (`app/routes/app.inbox.tsx`)
- **Out of scope:** Support page (`app/routes/app.support.tsx`) — not needed for now

## Architecture

### New component: `app/components/RichDraftEditor.tsx`

Encapsulates the TipTap editor instance, the toolbar, and the `EditorContent` zone. Extracted into its own file to keep `app.inbox.tsx` readable.

```ts
interface RichDraftEditorProps {
  content: string;       // HTML string (draft generated by AI)
  onChange: (html: string) => void;  // called on every change (hooks into existing autosave)
  readOnly?: boolean;    // true for archived versions
}
```

The `DraftBlock` in `app/routes/app.inbox.tsx` replaces its `<s-text-area>` with `<RichDraftEditor>`. All surrounding logic (To/Subject/CC fields, version history navigation, Regenerate button, autosave debounce) remains unchanged.

## Toolbar

Displayed above the editor zone, hidden when `readOnly={true}`.

```
B  I  U  |  ≡  1.  |  «  🔗  |  ✕
```

| Button | Action | TipTap extension |
|--------|--------|-----------------|
| **B** | Bold (toggle) | `starter-kit` |
| *I* | Italic (toggle) | `starter-kit` |
| U | Underline (toggle) | `@tiptap/extension-underline` |
| ≡ | Bullet list | `starter-kit` |
| 1. | Ordered list | `starter-kit` |
| « | Blockquote | `starter-kit` |
| 🔗 | Insert/edit link | `@tiptap/extension-link` |
| ✕ | Clear formatting | `unsetAllMarks()` + `clearNodes()` |

Active buttons show a distinct visual state (highlighted background, matching existing `ui-pill` style).

The link button opens a small inline popover to enter the URL. If the cursor is already on a link, the button removes it.

## Data Flow

### LLM → display

1. LLM generates a Markdown draft (prompt updated to request Markdown output with light formatting: `**bold**`, `-` lists, etc.)
2. Server converts Markdown → HTML using `marked` before storing/returning the draft
3. `DraftBlock` receives the HTML string as `initialBody`
4. TipTap loads and renders the HTML with formatting

### Agent editing → autosave

1. Agent edits in TipTap → `onChange(html)` fires on each change
2. `DraftBlock` receives the HTML string — same as it received plain text before
3. Existing 800ms debounce autosave is triggered unchanged
4. HTML is stored in the existing `body` column (no DB schema change)

### Copy-paste to Gmail

The agent copies the draft body. Gmail accepts HTML on paste and preserves formatting natively.

## Version History

Archived versions (read-only) currently display plain text in a read-only textarea. These are replaced by a rendered `div` using a sanitized HTML renderer. HTML must be sanitized with **DOMPurify** before rendering to prevent XSS — this is already done elsewhere in the project for email bodies. No toolbar is shown in read-only mode.

## Security

All HTML rendered outside of TipTap (archived versions, read-only display) must pass through DOMPurify before being injected into the DOM. TipTap's own editor enforces its own schema and is safe by design for editor content. HTML stored in the database comes from the LLM output (converted from Markdown) and from the TipTap editor — both are sanitized before display.

## Dependencies

| Package | Purpose | Notes |
|---------|---------|-------|
| `@tiptap/react` | React integration | Core |
| `@tiptap/pm` | ProseMirror peer dep | Required by TipTap |
| `@tiptap/starter-kit` | Bold, Italic, Lists, Blockquote, etc. | Covers most features |
| `@tiptap/extension-underline` | Underline | Not in starter-kit |
| `@tiptap/extension-link` | Hyperlinks | Not in starter-kit |
| `marked` | Markdown → HTML conversion (server-side) | Add only if not already present |
| `dompurify` | HTML sanitization for read-only display | Already used in inbox for email bodies |

No external CSS to import. Toolbar and editor zone are styled using the project's existing CSS tokens (`app/components/ui/tokens.css`).

## What Does Not Change

- DB schema — `body` column already stores text, HTML is just longer text
- Autosave mechanism — debounce logic in `DraftBlock` is unchanged
- To/Subject/CC/BCC fields
- Version history navigation UI (prev/next arrows, version count)
- Regenerate draft button and AI refine section
- All other routes and components
